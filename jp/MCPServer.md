# TRIGGERcmd MCPサーバー — ユーザーガイド

TRIGGERcmd MCPサーバーを使うと、Claude、ChatGPT、または
[Model Context Protocol](https://modelcontextprotocol.io) に対応したあらゆる
AIアシスタントが、あなたのTRIGGERcmdコマンドを認識し、普段どおりの会話だけで
あなたのコンピューター上で実行できるようになります。たとえば、アシスタントに
「オフィスのパソコンで何が実行できる?」「地下室の照明を消して」のように尋ねる
だけで、適切なTRIGGERcmdコマンドを見つけ出し、代わりに実行してくれます。

すべてのTRIGGERcmdアカウントですでに動作しているため、利用するためにあなたの
コンピューターに何かをインストールする必要はありません。

---

## AIアシスタントを接続する

接続方法は、お使いのAIツールが何に対応しているかによって2通りあります。

### オプション1 — Streamable HTTP(推奨)

「リモートMCPサーバー」や「コネクタ」に対応した最近のAI製品の多くは
(Claude.aiやChatGPTなど)、インターネット経由で直接TRIGGERcmdと通信できます。
AIツールがMCPサーバーのURLを尋ねてきたら、次のURLを入力してください。

```
https://www.triggercmd.com/mcp
```

このサーバーは標準のOAuth2に対応しているため、コネクタとして追加すると、
AIツールが自動的にTRIGGERcmdのサインイン/認可ページを開きます。一度承認すれ
ば、アシスタントはあなたのTRIGGERcmdアカウントのコマンドを参照・実行できる
ようになります。

> **ChatGPTをお使いの方へ:** TRIGGERcmdはすぐに使えるアプリとしても提供
> されています。
> **https://chatgpt.com/apps/triggercmd/asdk_app_694c84d39cb881918c6d181ae69e33fc**
> からインストールし、TRIGGERcmdアカウントでサインインするだけです。URLや
> 設定は不要です。

### オプション2 — ローカルstdioサーバー

お使いのAIツールがローカルで動く(「stdio」)MCPサーバーにしか対応していない
場合 — たとえば一部のClaude Desktopの構成など — は、代わりに次のコンパニオン
プロジェクトを使ってください。

**https://github.com/rvmey/triggercmd-mcp-stdio**

これはあなたのコンピューター上で小さなプロセスとして動作し、TRIGGERcmd
アカウントのトークンでサインインして、以下で説明するものとまったく同じ
ツール群を提供します。

---

## 接続後にアシスタントができること

接続すると、アシスタントはあなたのTRIGGERcmdアカウントに限定されたツールに
アクセスできるようになります — あなたが持つすべてのコンピューターと
コマンドを認識し、代わりに実行できます。ツールには3種類あります。

### `list_commands`

「どんなコマンドがあるか?」を尋ねるツールです。あなたのすべてのコンピュー
ターにあるコマンドを、次の情報とともにすべて返します。

- どのコンピューター上にあるか
- その名前
- その音声トリガーフレーズ(設定されている場合)
- そのMCPツールの説明(設定されている場合 — 詳しくは下記の「コマンドごとの
  動的ツール」を参照)

「TRIGGERcmdのコマンドは何がある?」「オフィスのPCで何が実行できる?」のよう
に尋ねたとき、アシスタントは何かを実行する前に、まず通常これを呼び出して
何が利用できるかを把握します。

### `run_command`

「何でも実行する」ための汎用ツールです。MCPツールの説明が設定されているか
どうかに関わらず、**どの**コンピューター上の**どんな**コマンドにも使えます。

アシスタントは次の項目を入力します。

- `computer` — そのコマンドを持つコンピューターの名前
- `command` — 実行するコマンドの名前
- `parameters` *(任意)* — パラメーターを受け付けるコマンドに渡す追加のテキスト

「ベッドルームのPCで『おやすみ』を実行して」「リビングのコンピューターで
Roku Playをトリガーして」のように、名前を指定して何かを実行するよう頼んだ
ときに、これが使われるのを目にすることが多いでしょう。

### コマンドごとの動的ツール

**MCP Tool Description**を入力したコマンドについては、TRIGGERcmdがそのコマンド
専用のツールを自動的に作成します。名前は `run_<コンピューター>_<コマンド>`
のような形式になります(小文字に変換され、スペースや特殊文字はアンダース
コアに置き換えられます)。

たとえば、「Office PC」というコンピューター上の「Play Music」というコマンド
は、アシスタントには `run_office_pc_play_music` という名前のツールとして表示
され、その説明にはMCP Tool Descriptionに入力したテキスト — たとえば
「オフィスのスピーカーで仕事用のプレイリストを再生します」のような内容
が使われます。

**これが役立つ理由:** コマンドに明確で具体的な説明を持つ専用ツールを与え
ることで、アシスタントは次のことができるようになります。

- 短い名前や汎用的な名前から推測するのではなく、*正しい*コマンドを素早く
  見つけて選ぶ
- コマンドのパラメーターをどのように整形すればよいか、いつ使うべきかを
  把握する
- 実行する前に、それが何をしようとしているのかを平易な言葉であなたに説明
  する

そのコマンドで**Allow Params**もオンになっている場合、その動的ツールは
`run_command` と同様に、任意の `parameters` 値を受け付けます。Allow Params
がオフの場合、その動的ツールは入力を受け付けず、単純にコマンドを実行する
だけです。

## コマンドに専用のツールを持たせる

コマンドをAIアシスタントに専用の名前付きツールとして表示させるには:

1. コマンドの設定を開きます(ローカルのTRIGGERcmdエージェントの
   `commands.json`、またはそのコマンドを管理している場所)。
2. **MCP Tool Description**に、そのコマンドが何をするか、および/または
   いつ使うべきかを説明する短く明確な一文を入力します — たとえば「地下の
   照明をすべて消す」「Plexサーバーを再起動する」など。
3. *(任意)* アシスタントからのテキストをコマンドのパラメーターとして
   受け付けたい場合は、**Allow Params**をオンにします。
4. 保存してエージェントを同期させます。MCPツールの説明が設定された
   コマンドの横には、TRIGGERcmdのコマンド一覧に🔧アイコンが表示されます。

次回アシスタントがツール一覧を取得すると、そのコマンド専用のツールが、
あなたが書いたとおりの説明とともに表示されます。

### 良い説明文を書くためのヒント

- *何が*起きるかを書きましょう。「地下の照明を点ける、または明るさを
  調整する。パラメーターには50のような明るさのパーセンテージを指定でき
  る」のように書き、単に「照明」とだけ書かないようにします。
- 短くまとめましょう — 一文か二文で十分です。アシスタントはツールを
  使うかどうかを判断するたびに、この説明を読みます。
